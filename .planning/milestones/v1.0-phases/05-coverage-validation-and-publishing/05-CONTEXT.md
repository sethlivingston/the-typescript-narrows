<decisions>
## Implementation Decisions

### Traceability matrix
- Lives at `docs/TRACEABILITY.md` (top-level docs, visible to external consumers)
- Minimal columns: Opinion ID | Skill section | Lint rule(s) | Status (covered/skill-only/gap)
- Generated by a script that reads opinion file frontmatter (lint field, skill references) — stays in sync automatically
- "Covered" for [both]-tagged opinions means: skill reference file section EXISTS and lint rule is either inherited from typescript-eslint strict-type-checked OR explicitly configured in our strict preset
- "Covered" for [skill-only]-tagged opinions means: corresponding section exists in a skill reference file

### Coverage gap handling
- Gaps in [both]-tagged opinions block publishing — must be resolved first
- Fix strategy is case-by-case: if a rule exists in typescript-eslint, add config to preset; if no rule exists and building one is impractical, reclassify to [skill-only] with documented rationale
- All 59 opinions validated — including the 22 skill-only opinions (confirm each has a skill reference file section)

### npm publishing
- Version 0.9.0 for first publish (bump from current 0.1.0)
- Unscoped package name: `eslint-plugin-typescript-narrows` (current)
- Add `eslint-plugin/README.md` with install instructions, config example, rule list, and link to skill
- Manual publish (npm publish from local) for v1 — CI/CD later
- MIT LICENSE file in `eslint-plugin/` (per-package for standalone publishability)

### Skill deployment
- Published to registry (GitHub repo reference pattern)
- Add `skill/the-typescript-narrows/README.md` — concise, no duplication with SKILL.md (link to SKILL.md for details, focus README on "what is this" and "how to install")
- MIT LICENSE file in `skill/the-typescript-narrows/`

### Licensing
- MIT LICENSE at repo root (expected on GitHub)
- MIT LICENSE in each publishable package (eslint-plugin/, skill/the-typescript-narrows/)
- Three LICENSE files total — root + 2 packages

### Root README
- Root README.md as the project landing page: what The TypeScript Narrows is, links to both artifacts, how they relate

### Claude's Discretion
- Traceability generation script implementation (Node.js, shell, etc.)
- README content structure and wording
- Exact format of gap closure documentation
- Whether to add npm `prepublishOnly` scripts or publish checks
- Package.json metadata (description, keywords, repository, homepage fields)
