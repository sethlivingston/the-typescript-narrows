  <action>
Create `scripts/generate-traceability.mjs` -- a zero-dependency Node.js ESM script.

**Data sources (3):**
1. `docs/opinions/*.md` (exclude INDEX.md and README.md) -- parse YAML frontmatter between `---` delimiters to extract: `id`, `title`, `severity`, `enforcement`, `lint.type`, `lint.rule`
2. `eslint-plugin/src/configs/strict.ts` -- regex-extract all rule names from lines matching `'rule-name': ` pattern (captures ESLint core, @typescript-eslint, import/, and typescript-narrows/ rules)
3. `skill/the-typescript-narrows/references/*.md` -- for each file, collect content to check if an opinion's id or title substring appears

**Frontmatter parser:** Split file content on `/^---$/m`, take the second segment, parse line-by-line:
- Top-level: `key: value` lines (strip quotes)
- Nested lint block: lines starting with whitespace under `lint:` key, parse `type: ...` and `rule: ...`
- Handle missing lint field (skill-only opinions)

**Rule extraction from strict.ts:** Match all lines with pattern `/'([^']+)':\s/` or `/"([^"]+)":\s/` to collect rule names into a Set.

**Skill coverage check:** For each opinion, search all reference file contents for the opinion `id` (kebab-case) converted to a regex pattern. Also check for the opinion title text. An opinion is "skill-covered" if its id or title appears in any reference file.

**Validation logic per opinion:**
- `enforcement: both` + `lint.type: existing` --> PASS if lint.rule is in strict.ts rule set AND skill-covered
- `enforcement: both` + `lint.type: custom` --> PASS if `typescript-narrows/{id}` is in strict.ts rule set AND skill-covered
- `enforcement: skill-only` --> PASS if skill-covered AND lint.rule is NOT in strict.ts rule set
- Anything else --> GAP

**Special cases:**
- `no-const-enum`: lint.type is `custom` but enforced as part of `ban-enums` rule. The script should check for `typescript-narrows/ban-enums` in strict.ts (the opinion frontmatter documents this).
- `exhaustive-discrimination` and `exhaustive-switch` may both map to `@typescript-eslint/switch-exhaustiveness-check`.
- `naming-convention` and `no-hungarian-notation` both map to `@typescript-eslint/naming-convention`.
- `no-unsafe-return` maps to `@typescript-eslint/no-unsafe-return` but also check `@typescript-eslint/no-unsafe-assignment` is present.

**Output format for docs/TRACEABILITY.md:**
```
# Traceability Matrix

Generated: {date} | Opinions: 59 | Covered: XX | Skill-only: XX | Gaps: XX

## Coverage

| Opinion ID | Title | Severity | Enforcement | Skill Reference | Lint Rule(s) | Status |
|------------|-------|----------|-------------|-----------------|--------------|--------|
| {id} | {title} | {severity initial} | {enforcement} | {reference filename or --} | {lint rule or --} | {covered|skill-only|gap} |

## Gaps

{table of gaps or "(none)"}
```

**CLI behavior:**
- No flags: generate docs/TRACEABILITY.md, print summary to stdout
- `--validate` flag: generate + exit 1 if any gaps exist, exit 0 if clean
- Print gap details to stderr when gaps found

**Path resolution:** Use `import.meta.url` with `fileURLToPath` to resolve paths relative to the script location (script is at `scripts/`, opinions at `../docs/opinions/`, etc.).
